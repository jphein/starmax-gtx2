Investigated & closed

Chased to a definitive answer — a documented hardware limit, or (for custom firmware) a proof on real hardware.

No path
Raw accelerometer stream

The LIS2DH12's raw XYZ is chip-internal only; stock firmware never exposes it over BLE. Derived step counts are pollable; raw motion is not.

No path
Live push / streaming

No push channel — every live value is a polled request. Poll fast for near-real-time; true streaming doesn't exist.

No path
GPS route / polyline

Two GPS-locked walks stored no track; the watch shows no map. GPS drives distance & pace only — the route is never retained.

Not on watch
Blood pressure

No BP category exists in the watch's health surface — the dashboard card has no sensor behind it on this device.

Not on watch
Body energy

Not among the categories the watch emits ({0,1,2,3,4,5,7}) — another dashboard card with no data source here.

Proven
Custom firmware — flashed & running

Repacked stock v1.0.3 with a marker string, recomputed its CRCs, and flashed it onto a spare over BLE with our own from-scratch tool. The watch CRC-verified the whole image (D4 ack d40000), committed, rebooted, and boots it — v1.0.3 confirmed on-device. There is no signature check: CRC-16 integrity only. Scope: guaranteed-boot images onto a designated spare; a wired un-brick path for a bad image is still unproven, so the flasher refuses anything but a verified allowlist entry pre-teardown.
